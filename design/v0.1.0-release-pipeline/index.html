    Neuralgentics v0.1.0 — Binary Release Pipeline Design¶

    Author: boomerang-architect (deepseek-v4-pro:cloud)
    Date: 2026-06-04
    Status: Design Complete
    Chain ID: b3302667-dc43-4f22-b1b2-a4bc4832a4fb
    Parent: v4-roll-your-own-app-FINAL.md

    1. Overview¶

    Neuralgentics v0.1.0 ships as pre-built binaries distributed via GitHub Releases. Users download a tarball and run install.sh — no source build required. A separate container-compose path exists for users who prefer "just give me an image."

    Design Goals: 1. Zero source builds for end users — build server does all compilation 2. Five target platforms covered (6 with Windows/arm64 fallback) 3. Single archive per target containing TUI binary + Go backend + Python sidecar source + install script 4. GitHub Actions matrix build on v* tag push 5. Container image option via podman-compose (ghcr.io) 6. "Hack the Planet" aesthetic — 1990s hacker movie vibe

    Design Anti-Goals: - No NPM package release (bun compile produces standalone binary) - No go install or source-only install option for end users - No auto-update (deferred to v0.2.0) - No Windows code signing (open-source CLI tool, not needed) - No Homebrew/cargo/choco package managers (deferred)

    2. Single Binary Archive Layout¶

    2.1 Per-Target Archive Structure¶

    2.3 Build Targets Matrix¶

    # Target GOOS/GOARCH Bun --target Runner OS Archive Notes
    1 linux-amd64 linux/amd64 bun-linux-x64 ubuntu-latest .tar.gz Primary target
    2 linux-arm64 linux/arm64 bun-linux-arm64 ubuntu-24.04-arm .tar.gz Raspberry Pi, AWS Graviton
    3 darwin-arm64 darwin/arm64 bun-darwin-arm64 macos-latest (M1/M2) .tar.gz Apple Silicon only
    4 windows-amd64 windows/amd64 bun-windows-x64 windows-latest .zip Standard Windows
    5 windows-arm64 windows/arm64 N/A (fallback) windows-latest .zip ⚠️ No standalone compile. Bundle source + require Bun runtime.

    Windows/arm64 caveat: bun build --compile does not support bun-windows-arm64 as of Bun v1.3.x. The Go backend cross-compiles fine (GOOS=windows GOARCH=arm64 CGO_ENABLED=0), but the TUI must ship as a bun bundle (.js files) rather than a standalone executable. The Windows ARM64 install script instructs the user to install Bun to run neuralgentics. This is tracked as Risk #1 (see §9).

    2.4 Python Sidecar Strategy¶

    The sidecar venv is ~3.5 GB (torch 2.3 GB + sentence-transformers ~1 GB + grpcio ~40 MB). It cannot be shipped inside the binary archive. Two paths are offered:

    Path A — Container (recommended, "quick start"): 

    # Single command, everything pre-built
podman pull ghcr.io/{owner}/neuralgentics-sidecar:latest
neuralgentics compose up
    The install script detects podman or docker and offers this path. If the user has neither, it falls back to Path B.

    Path B — Source venv (fallback, requires Python 3.11+): 

    # Creates ~/.neuralgentics/sidecar-venv/
neuralgentics sidecar install   # uses uv (faster) or pip
    This is the same as the current source-build approach but scoped to the sidecar only. The Go backend and TUI are still pre-built binaries.

    3. GitHub Actions Release Workflow¶

    3.1 Trigger¶

    on:
  push:
    tags: ['v*']

    On git tag v0.1.0 && git push origin v0.1.0, the workflow runs the full matrix build.

    3.2 Workflow YAML¶

    name: Release

on:
  push:
    tags: ['v*']

permissions:
  contents: write
  packages: write
  attestations: write
  id-token: write

jobs:
  # ── Job 1: Build binaries for all 5 targets ─────────────────────────────
  build:
    name: Build ${{ matrix.target }}
    runs-on: ${{ matrix.runner }}
    strategy:
      fail-fast: false  # one target failing doesn't block others
      matrix:
        include:
          # ── Linux x86_64 (ubiquitous) ──
          - target:           linux-amd64
            runner:           ubuntu-latest
            goos:             linux
            goarch:           amd64
            bun_target:       bun-linux-x64
            archive_fmt:      tar.gz

          # ── Linux ARM64 (Graviton, RPi 4/5 64-bit) ──
          - target:           linux-arm64
            runner:           ubuntu-24.04-arm
            goos:             linux
            goarch:           arm64
            bun_target:       bun-linux-arm64
            archive_fmt:      tar.gz

          # ── macOS ARM64 (Apple Silicon only) ──
          - target:           darwin-arm64
            runner:           macos-latest
            goos:             darwin
            goarch:           arm64
            bun_target:       bun-darwin-arm64
            archive_fmt:      tar.gz

          # ── Windows x86_64 ──
          - target:           windows-amd64
            runner:           windows-latest
            goos:             windows
            goarch:           amd64
            bun_target:       bun-windows-x64
            archive_fmt:      zip

          # ── Windows ARM64 (Snapdragon X, Surface Pro X) ──
          - target:           windows-arm64
            runner:           windows-latest
            goos:             windows
            goarch:           arm64
            bun_target:       none  # no standalone compile target exists
            archive_fmt:      zip

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Go 1.24
        uses: actions/setup-go@v5
        with:
          go-version: '1.24'

      - name: Setup Bun 1.3
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: '1.3'

      - name: Go module cache
        uses: actions/cache@v4
        with:
          path: |
            ~/go/pkg/mod
            ~/.cache/go-build
          key: go-${{ runner.os }}-${{ hashFiles('**/go.sum') }}
          restore-keys: go-${{ runner.os }}-

      # ── Build Go backend ──────────────────────────────────────────
      - name: Build Go backend (${{ matrix.goos }}/${{ matrix.goarch }})
        env:
          GOOS: ${{ matrix.goos }}
          GOARCH: ${{ matrix.goarch }}
          CGO_ENABLED: 0
        run: |
          cd packages/backend-go
          go build -ldflags="-s -w -X main.version=${GITHUB_REF_NAME#v}" \
            -o neuralgentics-backend ./cmd/backend/
          # On Windows, rename to .exe
          if [ "${{ matrix.goos }}" = "windows" ]; then
            mv neuralgentics-backend neuralgentics-backend.exe
          fi
        shell: bash

      # ── Build TUI (standalone binary) ─────────────────────────────
      - name: Build TUI (${{ matrix.target }})
        if: matrix.bun_target != 'none'
        run: |
          cd packages/tui
          bun install --frozen-lockfile
          bun build --compile \
            --target=${{ matrix.bun_target }} \
            --outfile=dist/neuralgentics \
            src/index.ts
          # On Windows, rename to .exe
          if [ "${{ matrix.goos }}" = "windows" ]; then
            mv dist/neuralgentics dist/neuralgentics.exe
          fi
        shell: bash

      # ── Build TUI (bundle, no compile — Windows ARM64 fallback) ───
      - name: Build TUI — bundle fallback (${{ matrix.target }})
        if: matrix.bun_target == 'none'
        run: |
          cd packages/tui
          bun install --frozen-lockfile
          bun build src/index.ts --outdir=dist --target=bun
          # Copy bun binary alongside for runtime
          cp "$(which bun)" dist/bun
        shell: bash

      # ── Assemble archive ─────────────────────────────────────────
      - name: Assemble archive
        run: |
          mkdir -p dist/neuralgentics/bin dist/neuralgentics/lib/embedding-sidecar

          # Go backend
          cp packages/backend-go/neuralgentics-backend* dist/neuralgentics/bin/

          # TUI binary (or bundle)
          cp packages/tui/dist/neuralgentics* dist/neuralgentics/bin/ 2>/dev/null || true
          # If bundle fallback (Windows ARM64), copy entire dist/ 
          if [ ! -f dist/neuralgentics/bin/neuralgentics ]; then
            cp -r packages/tui/dist/* dist/neuralgentics/bin/
          fi

          # Python sidecar source (no venv — users install deps)
          cp -r packages/memory/cmd/embedding-sidecar/*.py \
                packages/memory/cmd/embedding-sidecar/embedding_sidecar/ \
                packages/memory/cmd/embedding-sidecar/proto/ \
                packages/memory/cmd/embedding-sidecar/requirements.txt \
                dist/neuralgentics/lib/embedding-sidecar/

          # Install script
          cp scripts/install.sh dist/neuralgentics/install.sh
          chmod +x dist/neuralgentics/bin/*
          chmod +x dist/neuralgentics/install.sh

          # Create archive
          cd dist
          ARCHIVE_NAME="neuralgentics-${GITHUB_REF_NAME#v}-${{ matrix.target }}"
          if [ "${{ matrix.archive_fmt }}" = "tar.gz" ]; then
            tar -czf "${ARCHIVE_NAME}.tar.gz" neuralgentics/
            echo "archive=${ARCHIVE_NAME}.tar.gz" >> $GITHUB_OUTPUT
          else
            zip -r "${ARCHIVE_NAME}.zip" neuralgentics/
            echo "archive=${ARCHIVE_NAME}.zip" >> $GITHUB_OUTPUT
          fi
        shell: bash
        id: archive

      # ── Generate SHA256 for this archive ──────────────────────────
      - name: Generate SHA256
        run: |
          cd dist
          sha256sum "${{ steps.archive.outputs.archive }}" \
            >> checksum-${{ matrix.target }}.txt
        shell: bash

      # ── Upload as build artifact (for checksum aggregation) ──────
      - name: Upload archive
        uses: actions/upload-artifact@v4
        with:
          name: neuralgentics-${{ matrix.target }}
          path: dist/${{ steps.archive.outputs.archive }}
          retention-days: 1

      - name: Upload checksum
        uses: actions/upload-artifact@v4
        with:
          name: checksum-${{ matrix.target }}
          path: dist/checksum-${{ matrix.target }}.txt
          retention-days: 1

  # ── Job 2: Assemble checksums + create GitHub Release ────────────────
  release:
    name: Create GitHub Release
    runs-on: ubuntu-latest
    needs: [build]
    if: success()  # only if all builds pass

    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        with:
          path: artifacts/

      - name: Merge checksums
        run: |
          cat artifacts/checksum-*/*.txt > checksums.txt
          cat checksums.txt  # log for verification

      - name: Move archives to staging
        run: |
          mkdir -p staging
          find artifacts/neuralgentics-* -type f \( -name '*.tar.gz' -o -name '*.zip' \) \
            -exec cp {} staging/ \;
          cp checksums.txt staging/
          ls -lh staging/

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          name: "Neuralgentics v${{ github.ref_name }}"
          body_path: CHANGELOG.md
          files: staging/*
          generate_release_notes: true
          fail_on_unmatched_files: true

  # ── Job 3: Build and push container images ──────────────────────────
  containers:
    name: Build & Push Container Images
    runs-on: ubuntu-latest
    needs: [build]
    if: success()

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build & Push PostgreSQL + pgvector
        uses: docker/build-push-action@v6
        with:
          context: .
          file: containers/Dockerfile.postgres
          push: true
          tags: |
            ghcr.io/${{ github.repository_owner }}/neuralgentics-postgres:${{ github.ref_name }}
            ghcr.io/${{ github.repository_owner }}/neuralgentics-postgres:latest

      - name: Build & Push embedding sidecar
        uses: docker/build-push-action@v6
        with:
          context: .
          file: containers/Dockerfile.sidecar
          push: true
          tags: |
            ghcr.io/${{ github.repository_owner }}/neuralgentics-sidecar:${{ github.ref_name }}
            ghcr.io/${{ github.repository_owner }}/neuralgentics-sidecar:latest

      - name: Build & Push Go backend
        uses: docker/build-push-action@v6
        with:
          context: .
          file: containers/Dockerfile.backend
          push: true
          tags: |
            ghcr.io/${{ github.repository_owner }}/neuralgentics-backend:${{ github.ref_name }}
            ghcr.io/${{ github.repository_owner }}/neuralgentics-backend:latest

    4.2 Key Changes from Current install.sh (667 lines → ~400 lines)¶

    Current (source-build) New (binary download) Reason
    check_deps bun go python3 podman zig No deps required Binaries are pre-built
    ensure_source() — clones repo if piped Downloads .tar.gz from GitHub Releases No git clone needed
    build_backend() — go build from source Extracts pre-built binary from archive Build server does this
    install_tui_deps() — bun install && rsync TUI is standalone binary (bun build --compile) No Bun runtime needed

    4.3 WSL Detection¶

    detect_wsl() {
    # Method 1: Check kernel release for Microsoft string
    # WSL2 kernel: "5.15.153.1-microsoft-standard-WSL2"
    # WSL1: uses "Microsoft" (capital M) in uname output
    local kernel_release=""
    if [[ -f /proc/sys/kernel/osrelease ]]; then
        kernel_release="$(cat /proc/sys/kernel/osrelease 2>/dev/null)"
    fi

    # Method 2: Check for WSL_INTEROP env var (WSL2 only)
    local wsl_interop="${WSL_INTEROP:-}"

    # Method 3: Check uname for Microsoft
    local uname_microsoft=""
    if uname -r 2>/dev/null | grep -qi microsoft; then
        uname_microsoft="true"
    fi

    if [[ "$kernel_release" =~ [Mm]icrosoft || -n "$wsl_interop" || -n "$uname_microsoft" ]]; then
        log "⚠ Detected WSL — Windows Subsystem for Linux"
        log "  • Linux binaries work inside WSL"
        log "  • Windows PATH may conflict: add ~/.local/bin to your WSL shell rc, not Windows PATH"
        log "  • For native Windows install, run install.ps1 from a PowerShell terminal"
        return 0
    fi
    return 1
}

    4.4 Fish Shell PATH¶

    add_to_path() {
    local current_shell
    current_shell="$(basename "${SHELL:-bash}")"

    case "$current_shell" in
        fish)
            # fish_add_path is the canonical way — handles dedup and universal variables
            if ! fish -c "fish_add_path --check $BIN_LINK_DIR" 2>/dev/null; then
                fish -c "fish_add_path $BIN_LINK_DIR"
                log "Added $BIN_LINK_DIR to fish PATH"
            fi
            ;;
        zsh)
            local rc="${ZDOTDIR:-$HOME}/.zshrc"
            grep -q "$BIN_LINK_DIR" "$rc" 2>/dev/null || \
                printf '\n# neuralgentics\nexport PATH="%s:$PATH"\n' "$BIN_LINK_DIR" >> "$rc"
            ;;
        bash)
            local rc="$HOME/.bashrc"
            grep -q "$BIN_LINK_DIR" "$rc" 2>/dev/null || \
                printf '\n# neuralgentics\nexport PATH="%s:$PATH"\n' "$BIN_LINK_DIR" >> "$rc"
            ;;
        ash|sh)
            local rc="$HOME/.profile"
            grep -q "$BIN_LINK_DIR" "$rc" 2>/dev/null || \
                printf '\n# neuralgentics\nexport PATH="%s:$PATH"\n' "$BIN_LINK_DIR" >> "$rc"
            ;;
        *)
            warn "Unknown shell: $current_shell — add manually: export PATH=\"$BIN_LINK_DIR:\$PATH\""
            ;;
    esac
}

    9. Top 3 Risks and Mitigations¶

    Risk 1: Windows ARM64 — No Standalone TUI Binary¶

    Severity: Medium
    Likelihood: Certain (as of Bun v1.3, no bun-windows-arm64 target exists)

    Mitigation: - Ships TUI as bun build output bundle (.js files), not standalone executable. - install.ps1 detects Bun absence and prompts: winget install OpenJS.Bun - The TUI is invoked as bun run neuralgentics.js instead of ./neuralgentics.exe - Fallback: If Bun isn't available, the Go backend binary still works standalone via neuralgentics-backend.exe — the user can interact via JSON-RPC directly. - Monitor Bun releases for bun-windows-arm64 target (likely Bun 1.4+). When available, add to the matrix.

    Risk 2: Python Sidecar Venv Size (3.5+ GB)¶

    Severity: High
    Likelihood: Certain (torch CPU ~700 MB, GPU ~2.3 GB; sentence-transformers ~1 GB)

    Mitigation: - Container path is the primary recommendation. The install script detects podman/docker and strongly suggests the container path ("quick start"). If neither is available, it offers the venv path. - Container images pushed to GHCR. Users never build the sidecar image themselves — podman pull ghcr.io/.../neuralgentics-sidecar:latest gets a pre-built image. - Venv path uses uv for speed. uv pip install sentence-transformers is 10-50× faster than vanilla pip. Install time ~2-5 min vs 20+ min. - Sidecar is optional. The TUI starts in degraded mode without it — memory operations work, embeddings fall back to no-op. User can install sidecar later.

    Risk 3: macOS Code Signing + Gatekeeper¶

    Severity: Medium
    Likelihood: Probable (macOS Gatekeeper blocks unsigned binaries from running without user override)

    Mitigation: - The Go backend is a CLI binary, not a .app bundle — Gatekeeper applies only to bundles, not raw executables. However, xattr -d com.apple.quarantine may be needed on first run. - Install script runs: xattr -d com.apple.quarantine ~/.neuralgentics/bin/* 2>/dev/null || true - Future (v0.2.0): Consider Apple Developer ID signing. Requires $99/year Apple Developer Program membership. - For now: document the right-click → Open workaround in the install script output.

    10. Open Questions — Resolved¶

    Question Resolution
    How does bun --compile handle cross-platform? Works from any host for bun-linux-x64, bun-linux-arm64, bun-darwin-arm64, bun-windows-x64. No bun-windows-arm64 target exists yet — fallback to source bundle. macOS targets require macOS runner (M1 runner needed for bun-darwin-arm64).
    Should Python sidecar be in the binary or shipped separately? Separately. Shipping a 3.5 GB venv in the archive is infeasible. Container path is primary; source venv path is fallback.
    WSL1 vs WSL2 detection difference? Both detectable. WSL2 has WSL_INTEROP env var + microsoft in kernel release string. WSL1 has Microsoft (capital M) in uname -r or osrelease. The detection script handles both.
    Windows .exe signing? Skipped for v0.1.0. Open-source CLI tool, not a GUI app. Future: consider EV code signing certificate (~$300/year) if Windows users report SmartScreen issues.
    Auto-update mechanism? Deferred to v0.2.0. For v0.1.0, users re-run the install script to upgrade. The neuralgentics upgrade command checks the GitHub API for latest release and prints instructions.
