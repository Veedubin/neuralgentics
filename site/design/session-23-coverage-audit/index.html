    3. Top 10 Coverage Gaps¶

    Gap #1: store/memories.go — All CRUD operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: AddMemory, GetMemory, DeleteMemory, CountMemories, ListMemories, UpdateMemory, ContentExists, UpdateTrustFields, IncrementRetrievalCount
    • Why it matters: These are the most-called functions in the entire system. Every memory operation goes through this code. A bug here corrupts all data.
    • Test sketch: TestAddMemory_WithEmbedding — create a PostgresStore with mock pool, call AddMemory with a vector, verify INSERT was called. TestGetMemory_IncludeArchived — verify correct query is used based on includeArchived flag.

    Gap #2: store/memories.go — Relationship operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: CreateRelationship, DeleteRelationship, GetRelationships, GetRelationshipSummary, GetSupersessionChain, GetSuperseded
    • Why it matters: The knowledge graph and dialectic engine depend on these. A broken GetRelationships breaks contradiction detection.
    • Test sketch: TestGetRelationships_MalformedRow — insert a row with NULL in a non-nullable column, verify the error is logged (not silently dropped).

    Gap #3: store/memories.go — Entity operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: UpsertEntity, GetEntity, GetEntitiesByType, SearchEntities, CreateEntityRelationship, GetEntityRelationships, ResolveEntityGraph, InferenceChain
    • Why it matters: The KG subsystem is a core feature. InferenceChain BFS traversal is complex and untested.
    • Test sketch: TestInferenceChain_ShortestPath — create entities A→B→C, verify BFS finds A→C path.

    Gap #4: store/memories.go — Peer operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: AddPeer, GetPeer, ListPeers, UpdatePeerLastActive, ShareMemory, RevokeShareMemory, GetSharedMemories, GetPeerMemories
    • Why it matters: Multi-peer memory sharing is a key differentiator. Untested sharing logic could leak data.
    • Test sketch: TestShareMemory_RoundTrip — share a memory with a peer, verify GetSharedMemories returns it.

    Gap #5: store/memories.go — Thought chain operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: StartThoughtChain, AddThought, GetThoughtChain, GetRelatedChains, ReviseThought, BranchThought, PauseThoughtChain, ResumeThoughtChain, AbandonThoughtChain
    • Why it matters: The thought chain system is the core of the Boomerang Protocol. Untested branching/revision logic could lose reasoning context.
    • Test sketch: TestReviseThought_CreatesRevision — add a thought, revise it, verify revision has is_revision=true and revises_thought_id points to original.

    Gap #6: store/memories.go — Project chunk operations (5.5% coverage)¶

    • Package: memory/store
    • Untested functions: AddProjectChunk, DeleteChunksByPath, SearchChunks, GetFileChunksByPath
    • Why it matters: The project indexer depends on these. SearchChunks has dynamic SQL construction with path/file type filters — SQL injection risk.
    • Test sketch: TestSearchChunks_DynamicSQL — search with path and file type filters, verify correct WHERE clause construction.

    Gap #7: search/hybrid.go — RRF fusion logic (26.8% coverage)¶

    • Package: memory/search
    • Untested functions: rrfFuseMultiList, rrfMerge, autoQuery (dual-model RRF), gpuQuery, cpuQuery
    • Why it matters: The RRF fusion is the core of the search quality. A bug here produces wrong search results.
    • Test sketch: TestRRFFuseMultiList_ThreeLists — create 3 ranked lists with overlapping items, verify RRF scores are computed correctly and items are sorted by descending score.

    Gap #8: context_builder.go — All 5 exported functions (0% coverage)¶

    • Package: orchestrator
    • Untested functions: BuildContextPackage, StoreContextPackage, FetchContextPackage, StoreAgentWrapUp, FetchAgentWrapUp
    • Why it matters: This is the core orchestrator logic for building agent context packages. A bug here sends wrong context to agents.
    • Test sketch: TestBuildContextPackage_WithRelevantMemories — create a mock MemoryProvider that returns 3 memories, call BuildContextPackage, verify RelevantFiles and CodeSnippets are populated.

    Gap #9: broker.go — Call, HealthDeep, MatchIntent (55% coverage)¶

    • Package: broker
    • Untested functions: Call (access denied, server not found, no pipes), HealthDeep (context timeout, nil pipes), MatchIntent (no tools available, role filtering)
    • Why it matters: Call is the primary entry point for tool dispatch. A bug here blocks all MCP tool calls.
    • Test sketch: TestBroker_Call_AccessDenied — call a server the role doesn't have access to, verify ErrUnauthorized with available server hints.

    Gap #10: backend-go/main.go — JSON-RPC dispatch (32.3% coverage)¶

    • Package: backend/cmd
    • Untested functions: JSON-RPC method dispatch, error handling, signal handling, JSONBrokerCallResult (nil Result edge case)
    • Why it matters: This is the entry point for all TUI-to-backend communication. A bug here breaks the entire system.
    • Test sketch: TestJSONBrokerCallResult_NilResult — call a tool that returns nil result, verify the response is handled correctly.
